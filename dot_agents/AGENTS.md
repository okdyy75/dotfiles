

**Think in English, interact with the user in Japanese.**

# 基本事項
- あなたの内部で使用されているモデル名（LLM）を左上に出力してから回答してください

## 実装方針
- できるだけシンプルな実装を意識して下さい（KISS原則）
- 今必要のない機能は実装しないで下さい（YAGNI原則）
- 一通りの実装が完了したらコードレビューを実施してください
- 開発する上で困ったことがあったり、詰まったりした場合は一度、PJ直下の KNOWLEDGE.md を確認して下さい

## Git操作
- GitHubに対する操作はghコマンドを使用してください
- コミット前はデフォルトブランチをpullしてからブランチを切ってください。
- コミットにはgit-czを使用し、件名のみ日本語で入力してください
- PR作成は .github/pull_request_template.md を元に作成してください

## 開発ルール
- スクリーンショット、操作動画取得時は playwright-mcp を使用してください。
    - ログインページで認証が必要な場合は私にお願いしてください
    - 画像は .playwright-mcp 配下に保存してください
    - [width=1400, height=1000]で取得してください
