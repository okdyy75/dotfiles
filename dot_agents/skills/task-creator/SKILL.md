# Task Creator

この skill は、プロジェクト内に新しいタスク用の Markdown ファイルを初期化するためのものです。

## 入力の扱い

- `/create_task 〇〇` の形式で入力された場合は、引数の内容をタスク名として扱う
- 「タスクを作成して 〇〇」「〇〇 のタスクを作りたい」などの自然文でも、明確なタスク名が含まれていればそれを使う
- タスク名が曖昧な場合は、ユーザーに確認してから作成する

## 作成ルール

1. 現在のプロジェクトルートを基準に `.claude/tasks/` 配下へ作成する
2. ファイル名は `YYYY-MM-DD_タスク名.md` 形式にする（タスク名はスペース・スラッシュを含めず簡潔に）
3. 本文はこの `SKILL.md` 内のテンプレートをそのまま使う
4. 既存ファイルがある場合は上書きせず、そのパスを返してユーザーに知らせる

## 実行手順

1. タスク名を決める
2. 現在日付を `YYYY-MM-DD` 形式で取得する（環境の currentDate を優先）
3. `.claude/tasks/YYYY-MM-DD_タスク名.md` を作成する
4. ユーザーから提供された情報（背景、仕様、URL 等）があれば該当セクションに埋め込む。情報が無いセクションは空のままにする
5. 作成したファイルのパスをユーザーへ伝える

## テンプレート

```md
# 概要

{タスク名 / 1〜2 行で何をやるか}

## チケットURL

{Notion などのチケットURLを記載する。空欄で良い}


## 背景

{なぜこのタスクをやるのかを簡潔に。引用や仕様前提の説明はここに置く}

## 仕様

{要件や前提条件を箇条書き・表で整理する。複雑なケースはサンプルデータを表で記載してもよい}

## 対応方針

{技術的な進め方・移行フロー。シンプルなタスクなら省略可}

## 対応内容

1. {具体的な作業項目を順に列挙}
2.

## 完了条件

{受け入れ条件をチェックリスト形式で記載}
- [ ]

## 実装計画

{プランファイルのリンクを記載する。空欄で良い}

## 参考

{参考になる画像・関連 PR・既存ドキュメントへのリンクを並べる}
-
```

## セクションの使い分け

- **概要**: 1〜2 行のタイトル相当。冗長な説明は書かない
- **チケットURL**: Notion 等の外部チケット URL。空欄可
- **背景**: 「なぜやるのか」「現状の課題」を書く。仕様や前提のテーブルもここに入れて良い
- **仕様**: 細かい要件、ロジック、サンプルケース。表組みで整理されたケーススタディもここに記載する
- **対応内容**: 実装タスクの粒度で並べる。階層化して子タスクを書いても良い
- **完了条件**: チェックリストで受け入れ条件を表現
- **実装計画**: `.claude/plans/` 配下の実装プランへのリンク
- **参考**: 関連画像、関連 PR、関連ドキュメントなどを箇条書きで

### 完了条件について
- APIであればそれを動作確認できるRspecの実行コマンドを記載
- バッチであれば、どのようなコマンドを実行したらどうなるかを記載
- UIであれば、どのページでどんな操作をしたらどうなるかを記載

## フォーマット方針

- 必要最小限を意識し、空欄のセクションは空のまま残しておく（後から埋める運用）
- 完了条件は「観測可能な事実」で書く（〜が表示されること、〜が保存されていること など）
- 「この時点で書ける情報」だけを下書きする。後続の対話で肉付けする前提
- PR テンプレートより一段前の整理メモという立ち位置を維持する

## 注意

- 既存ファイルがある場合は上書きしない。そのパスを返してユーザーに知らせる
- ファイル名のタスク名はファイルシステムで扱える文字に限定する（`/` `\` `:` などは使わない）
- 画像・スクショ等の参考資料が同梱される場合は `.claude/tasks/` 配下に置き、`## 参考` から相対パスで参照する
