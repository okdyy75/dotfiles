# Knowledge

現在のセッションの気づき・学びを抽出し、プロジェクト直下の `KNOWLEDGE.md` を重複排除しながら追記・更新するスキル。

## 手順

### 1. 既存の KNOWLEDGE.md を読み込む

PJ直下の `KNOWLEDGE.md` を Read で読み込み、既存の見出し構成と項目を把握する。
ファイルが存在しない場合は新規作成する。

### 2. セッション内容を振り返る

現在のセッションの会話履歴・差分・実行ログから、再利用可能な知見だけを抽出する。

**抽出対象:**
- コーディング規約・命名規則・パターンに関する判断
- 環境構築・コマンド実行の注意点（direnv, docker compose, etc.）
- ライブラリ・フレームワーク固有の落とし穴
- レビューで指摘された設計指針
- 詰まった問題とその解決方針（fixそのものではなく汎用化できる方針）

**抽出しないもの:**
- 日付・セッションID・プロジェクト名などのメタ情報
- 今回タスク固有の作業ログや差分
- コード自体を読めばわかる事実
- 既存の `KNOWLEDGE.md` と重複する内容

### 3. 候補を箇条書きで列挙する

抽出した知見を以下の形式で並べ、ユーザーに見せて精査する：

```markdown
- [気づき・学び・指針]
- ...
```

各項目は短く具体的に書く。冗長な背景説明は避ける。

### 4. KNOWLEDGE.md に反映する

既存の見出し構成（例: `## コマンド実行`, `## 実装方針`, `## GraphQL / Ruby mutation パターン` など）に沿って Edit で追記する。

**ルール:**
- 既存と同じ趣旨の項目は追加しない（重複排除）
- 矛盾する古い情報があれば新しい情報で上書きする
- 適切な見出しが無い場合のみ新しい `##` セクションを追加する
- 箇条書きスタイルを維持する（既存ファイルに合わせる）

### 5. 完了を報告

更新した見出しと、追加・更新した項目の概要を日本語で報告する。

## 出力ルール

- メタ情報（日時、セッションID、プロジェクト名）は書かない
- タスク固有の実施ログは書かない
- 汎用化できる学びだけ残す
- 箇条書きは短く具体的に書く
